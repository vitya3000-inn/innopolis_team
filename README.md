# NewsMap

NewsMap — пет-проект, который показывает политические новости не бесконечной лентой заголовков, а несколькими темами и внутри них событиями. Одно событие — это смысловой «узел»: несколько статей из разных СМИ склеены вместе, к ним добавлены краткое резюме и карточки «как это подают разные издания». Идея MVP: на главном экране всегда три фиксированные темы (Иран, Украина, Израиль/Газа/Ливан), а не случайный набор рубрик.

Клиент — Expo / React Native (iOS, Android, веб). Сервер — Node.js на голом http (без Express). Тяжёлая работа (скачивание новостей, кластеризация, ИИ) делается на сервере; в приложение утекает только уже готовый JSON и публичные ключи (URL API, Supabase anon, Turnstile site key).
---

Общая схема «кто с кем говорит»
Периодически или по команде админа сервер запускает обновление ленты (ingestion): тянет статьи из NewsAPI, при желании переводит и обогащает текст, склеивает статьи в события, раскладывает по темам, при необходимости вызывает OpenAI.
Результат текущей ленты сохраняется в файл backend/data/store.json на машине с бэкендом.
Если включены Supabase и нужные таблицы, в базу пишется история прогонов и «снимки» данных — это нужно для архива по дню и части аналитики.
Приложение не ходит в NewsAPI и не держит секреты OpenAI: оно только делает HTTP GET к вашему API (/topics, события и т.д.). Параметр ?date=YYYY-MM-DD означает «дай не свежий store.json, а последний успешный прогон за этот календарный день UTC из Supabase».
Авторизация (если настроены URL и anon-ключ Supabase в клиенте) — через Supabase Auth; сессия в AsyncStorage. Если клиентский Supabase не настроен, приложение сразу открывает ленту без входа.
Экран админа позволяет дернуть полное обновление с сервера (POST /admin/refresh) — только для email из белого списка и с валидным JWT (в dev можно ослабить правила через env — это уже документировано отдельно).

### Стек

| Часть | Технологии |
|--------|------------|
| Клиент | Expo ~54, React Native, React Navigation |
| Сервер | Node.js, `http`, без Express |
| Данные | `backend/data/store.json` (текущая лента), опционально PostgreSQL через Supabase |
| Внешние API | NewsAPI, Google Translate (бесплатный endpoint), OpenAI Chat Completions |

Клиентское приложение
Навигация (App.tsx): стопки экранов Лента → Тема → Событие; при настроенном Auth сначала экран входа/регистрации.
Лента: темы, фильтр по категории, календарь архива (UTC), pull-to-refresh (только перечитывает готовые данные с API, не запускает пайплайн на сервере).
Тема и событие: список событий и детальная карточка с AI-текстом и мнением СМИ со ссылками.
Тема / светлая тема, тосты, шапка с аккаунтом для залогиненных.
Аналитика: при открытии ленты может писаться строка в app_visits (для админской статистики по периоду).
Пейвол (эксперимент): при авторизации считаются уникальные UTC-дни посещения актуальной ленты в feed_visit_days; после 7 таких дней без «подписки» в user_entitlements может показываться модалка оплаты (по сути задел под монетизацию).
Cloudflare Turnstile (если настроены ключи): проверка «не бот» на сервере для эндпоинта верификации; конфиг подтягивается из backend.
Базовый URL API задаётся через EXPO_PUBLIC_API_URL или extra в app.config.js; для телефона в той же сети часто нужен LAN IP ноутбука, а не localhost.

---

Backend: роль и основные файлы
server.js — HTTP-маршруты, health, отдача тем/событий, админские ручки, фоновый интервал обновления после старта.
pipeline.js — единая цепочка «статьи → события → темы → запись».
newsApi.js — запросы к NewsAPI, кэш, лимиты и задержки.
cluster.js — фикс-темы, эвристическая кластеризация, сборка структуры под UI.
articleLlmGroup.js — альтернативная склейка статей через OpenAI (включается флагом).
llmOpenAI.js — обогащение событий, кэш ответов на диске.
store.js — чтение/запись store.json.
supabasePersist.js / supabaseHistory.js — запись прогона и чтение архива по дате.
adminAuth.js — проверка JWT и списка админов для защищённых ручек.
config.js — все лимиты и переменные окружения в одном месте.
Секреты (NewsAPI, OpenAI, service role Supabase, Turnstile secret) должны жить только на сервере.


## Мобильное приложение

- **Навигация:** при настроенном Supabase Auth — сначала экран входа/регистрации, затем `Feed` → `Topic` → `Event` (`App.tsx`).
- **Главный экран** — темы, фильтр по категории, календарь архива; у вошедшего пользователя в шапке — email и выход.
- **Экран темы** — список событий; **экран события** — AI summary, факты, карточки СМИ со ссылками на оригиналы.
- **API-клиент** — **`src/services/newsApi.ts`**: базовый URL из **`EXPO_PUBLIC_API_URL`** / Expo extra / LAN.

---

Данные и «архивный день»
Актуальная лента — это содержимое store.json после последнего успешного refresh.
Архив — не «новости по дате публикации в источнике», а снимок того, что билд пайплайна записал в Supabase в выбранный UTC-день (последний прогон за этот день). Если прогона не было, API может вернуть «пустые» темы-заглушки с пометкой в метаданных.
Календарь в приложении выбирает день; на вебе возможен ручной ввод YYYY-MM-DD.
